## 产品架构关键决策

### Market（B2B交易）
- 不收佣金（暂时），先跑量
- 三类购买者：Sponsor（赞助商）/ Exhibitor（参展商）/ Attendee（观众）
- 同一账号多角色，统一Commerce流程
- 展会方在EventModuleSetup选模块 → 自动生成对应赞助产品目录 → 赞助商购买

### Sponsor产品逻辑
- 展会方自己配置本展会提供哪些赞助产品（不统一）
- SponsorBuilder提供矩阵视图（Excel式）和导购流程两种模式
- 核心盈利：赞助商收费抽成

### Live平台化（v43思路）
- Live = 通用模板，任何展会都能用
- 顶部展会选择器（横向滚动卡片：LIVE/SOON/ENDED状态）
- 展会方通过B2B Events后台发布活动，自动出现在Live里

### Auth
- 只用Google登录，不做邮箱密码
- 首次登录进入Onboarding（4步：选角色→填资料→选兴趣→确认）
- 登录后按账号角色自动跳转对应后台

## Let's Meet 产品逻辑

- Matchmaking是展会方在EventModuleSetup里选择开启的模块之一（和Awards/Showcase平级）
- 开启后展会方配置：时间槽/会议时长/房间数/AI matching开关
- 参会者：Browse参会者 → 发Request → 对方接受 → 进入日历
- 展会方：看全局匹配情况、房间分配、冲突检测
- EventModuleSetup_v2加了"GAME CONNECTION EXCLUSIVE" badge，区分平台特色功能

## 公开展会主页逻辑

- EventPublicPage是对外公开的（不是后台），类似Eventbrite详情页
- 包含：Hero/Stats/About/Featured Games/Speakers/Let's Meet区块/Awards/Sponsors/购票CTA
- 每个展会有唯一公开URL（gameconnection.com/events/indie-game-fest-2026）
- 展会方开启了哪些模块，公开页就显示对应区块
