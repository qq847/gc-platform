# Game Connection Platform — 完整设计思路
*整理自 2026-03-22 深夜对话*

---

## 背景

Game Connection 是一个有历史的线下游戏展会品牌，已破产清算。Qiang Qiang 正在用已有的数据资产、用户关系和品牌，重新做成一个线上平台。

---

## 核心定位

**游戏行业专属平台**——把线下展会的人脉、项目、资源搬到线上，同时保留并强化线下展会场景。

不是论坛，不是新闻站，是：
- 游戏行业版 LinkedIn（人与人连接）
- 项目展示与撮合平台（开发者↔发行商↔投资人）
- 展会工具提供商（赋能全球小型展会）

---

## 用户体系（三位一体 Profile）

每个使用者由三个维度构成完整画像：

| 实体 | 说明 |
|------|------|
| **Person** | 个人账号（开发者、发行商、投资人、PR等） |
| **Organization** | 所属单位（游戏工作室、发行公司、协会、媒体） |
| **Project** | 游戏项目（可关联多个Person和Organization） |

三者互相关联，形成完整的行业身份认证体系。

---

## 平台功能架构（Roadmap v2）

### 主页（左上角）
- **Feed** — 信息流主页，展示7大类内容
- 内容由平台用户和接入展会实时产生
- 最终要是动态生成的，不是静态图

### 核心用户功能（中间区域）
围绕 Person / Organization / Project 三个实体提供：

| 功能 | 说明 |
|------|------|
| **Display** | 游戏项目展示页，7000+项目待上传 |
| **Match** | Person ↔ Person 撮合配对（Matchmaking） |
| **Pitch** | 项目发行/融资对接 |
| **Market** | 市场功能（待细化） |

### B2B 展会赋能（右上角）
**Empowering** — 核心差异化功能，赋能全球小型展会：
- 小展会入驻平台，发布活动信息
- 利用平台 Buyer 数据库，帮小展会招募优质参展商
- **联合拉赞助**：多个小展会单独拉不到赞助，打包在一起，平台做中间人统一谈判
- 这块是重要收入来源，也是平台对小展会的核心价值

### Live 工具（线下展会入口）
- **定位**：大型展会期间的附属工具，提供导航、活动安排、参会者信息等
- **已验证**：GDC 2026 期间上线（GDC2026_v40.html），4000+ 用户使用，效果很好
- **战略意图**：每次大展会 = 一次大规模拉新机会，用 Live 工具把线下用户引入平台注册
- **计划**：标准化成模板，在 Gamescom、CJGC、TGS 等各大展会期间复用

## 开发 Milestones

**M1 — 基础骨架（1-2个月）**
张元琦主导：

- 项目对接功能
- 平台用户量和数据量到位后推出
